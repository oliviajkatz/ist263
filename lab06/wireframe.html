Mapping: wireframe regions → files

  • Header / Nav — implement in a shared header or each page's top (e.g., inside bio.html, skills.html, contact.html).
  • Main panels — map to semantic sections: hero, about (bio), skills (table), contact (form).
  • Footer — small site info placed in each page's bottom or included via a template.

HTML skeleton (copy into a page)

<header class="site-header">
  <nav> ... </nav>
</header>

<main>
  <section class="hero">...</section>
  <section id="about" class="panel">...bio content...</section>
  <section id="skills" class="panel">...skills table...</section>
  <section id="contact" class="panel">...contact form...</section>
</main>

<footer>...</footer>

Tip: keep your styles in styles.css and add class names that match the skeleton sections.
